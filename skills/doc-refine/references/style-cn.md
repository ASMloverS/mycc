# 中文文言文 Style

以文言重写。技术内容不丢。废话尽去。

## 核心规则

- 全文用文言文重写（lite 级别除外）。
- 四字格优先，短句优先。
- 代码块、API 名、变量名、命令、路径、错误消息保留原文不译。
- 有疑则存。

## 句法规则

- **省主语**：主语可推断时省之。例："系统将自动检测" → "自动检测" → "自检"。
- **判断句**：用"…也"。例："这是核心模块" → "核心模块也"。
- **被动**：用"为…所…"。例："被系统拒绝" → "为系统所拒"。
- **否定**：用"不/无/未/非"。例："不能使用" → "不可用"。
- **并列**：用"、"分隔。例："读取、解析、验证数据"。
- **因果**：用"故/是以/遂"。例："配置错误，所以启动失败" → "配置有误，故启动不成"。

## 强度

| 级别 | 变化 |
|------|------|
| lite | 白话精简，去客套铺垫，偶用文言词汇。完整语法 |
| full | 全文文言文。四字格优先，主语可省，判断句/被动句/否定句用文言式 |
| ultra | 极简文言。一字能尽不用两字，词组化，箭头化（X → Y） |

## 范例

**原文：**
> 本模块主要负责对用户输入的数据进行有效性验证，确保数据符合预定义的格式要求，如果验证失败则返回相应的错误信息。

**lite:** 验证用户输入是否符合预设格式。验证失败则返回错误信息。

**full:** 验用户输入，合预设格式则过之，不合则返报错。

**ultra:** 验输入格式。不合→报错。

---

**原文：**
> 为了提高系统的性能，我们需要对数据库的查询进行优化，主要是通过添加索引和减少不必要的联表查询来实现。

**lite:** 优化数据库查询以提升性能：添加索引，减少不必要的联表查询。

**full:** 欲提升效能，须优化数据库查询。法有二：一曰增索引，二曰减冗余联表。

**ultra:** 优化 DB 查询 → 提速。法：增索引、减联表。

---

**原文：**
> 连接池会复用已打开的数据库连接，而不是每个请求都新建连接，这样能减少握手开销。

**lite:** 连接池复用已打开的数据库连接，避免每次请求都新建，减少握手开销。

**full:** 连接池复用现成数据库连接，无须每求新建，遂省握手之耗。

**ultra:** 池 = 复用 DB 连接。免新建 → 省握手。
